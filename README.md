# ML-система для предсказания факта страховой выплаты

## О проекте

Это ML-система для обработки потоковых табличных данных на Python.
Она работает со страховыми данными и решает задачу **бинарной классификации**:

- `1` - была выплата по страховке;
- `0` - выплаты не было.

Для обучения используется колонка `CLAIM_PAID`, которая преобразована следующим образом, чтобы подходить под бинарную классификацию:
- `CLAIM_PAID > 0` => `target = 1`
- `CLAIM_PAID` пустой или `0` => `target = 0`

Общий пайплайн работы:
- исходный датасет режется на батчи;
- батчи по одному поступают в систему;
- каждый новый батч сохраняется в хранилище;
- по нему считаются метрики качества данных;
- данные очищаются;
- обновляется накопленное хранилище очищенных данных;
- модели переобучаются;
- лучшая модель сохраняется для дальнейшего использования.

---

## Что делает система

Проект покрывает основные этапы ML-конвейера:

1. **Сбор данных**
   - разбиение исходного файла на батчи;
   - эмуляция потока данных;
   - хранение сырых батчей;
   - расчет метапараметров батча.

2. **Анализ данных**
   - оценка качества данных;
   - сохранение метрик качества;
   - базовая очистка;
   - построение ассоциативных правил по бинарным признакам.

3. **Подготовка данных**
   - обработка пропусков;
   - обработка категориальных признаков;
   - обработка числовых признаков;
   - генерация простых признаков.

4. **Обучение моделей**
   - обучение модели на деревьях решений;
   - обучение нейронной сети;
   - выбор лучшей модели.

5. **Валидация**
   - временное разбиение train/valid;
   - расчет метрик качества;
   - хранение версий моделей и результатов.

6. **Обслуживание**
   - сериализация лучшей модели;
   - применение модели к новым данным;
   - формирование summary-отчета.

## Структура проекта

```text
├── config.json               # конфигурация проекта
├── requirements.txt          # требования по библиотекам
├── run.py                    # основной скрипт запуска
├── data/
│   ├── source/               # исходный датасет
│   ├── batches/              # батчи, полученные из исходного файла
│   ├── raw_storage/          # необработанные батчи
│   └── clean_storage/        # очищенные батчи
├── artifacts/
│   ├── models/               # сохраненные версии моделей
│   ├── metrics/              # метрики моделей
│   ├── reports/              # отчеты
│   └── predictions/          # результаты работы режима inference
└── src/
    ├── common.py             # начальная обработка директорий проекта
    ├── data_collection.py    # обработка данных
    ├── data_quality.py       # измерение качества данных
    ├── features.py           # работа с признаками в данных
    ├── modeling.py           # построение моделей (деревьев и mlp)
    ├── pipeline.py           # пайплайн работы
    └── reporting.py          # файл для составления summary
```
